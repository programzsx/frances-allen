# 手动打包指南

> 适用项目：frances-allen（知识库）
> 最后更新：2026-05-31

## 前置准备

- Windows 系统，已安装 Flutter SDK（`flutter --version` 可执行）
- JDK 17 在 `C:\Program Files\Java\jdk-17`
- 当前在项目根目录 `frances-allen/`

## 打包前确认清单

- [ ] Android 应用名设为「知识库」（`mobile/android/app/src/main/AndroidManifest.xml` 中 `android:label="知识库"`）
- [ ] 自定义图标已覆盖 `mobile/assets/kb.png`（用 `zsx-build-deploy/knowbase.png` 替换）
- [ ] `mobile/android/gradle.properties` 中有 `kotlin.compiler.execution.strategy=in-process`（修复 Windows JDK 访问权限错误 1920）
- [ ] `mobile/android/gradle/wrapper/gradle-wrapper.properties` 用腾讯镜像：`https://mirrors.cloud.tencent.com/gradle/gradle-8.14-all.zip`（国内下载加速，否则超时）

## 第一步：替换自定义图标

```bash
# 用 zsx-build-deploy 下的 knowbase.png 覆盖 assets/kb.png
cp zsx-build-deploy/knowbase.png mobile/assets/kb.png
```

## 第二步：复制项目到 Windows 临时目录

> **注意**：每次用新目录名（如 `temp_fa_mobile`、`temp_fa_mobile2`），不要复用。旧目录可能被 Gradle daemon 锁定，`rm -rf` 会 Permission denied。

```bash
rm -rf /mnt/c/temp_fa_mobile
cp -r mobile /mnt/c/temp_fa_mobile
```

## 第三步：安装依赖（Windows cmd）

```bash
cmd.exe /c "cd /d C:\temp_fa_mobile && flutter pub get"
```

## 第四步：生成自定义图标（Windows cmd）

> **不可跳过**。如果不执行这一步，APK 安装后显示的是默认 Flutter 图标，不是 `knowbase.png`。

```bash
cmd.exe /c "cd /d C:\temp_fa_mobile && flutter pub run flutter_launcher_icons"
```

预期输出：`✓ Successfully generated launcher icons`

## 第五步：构建 APK（Windows cmd）

> 这一步最耗时，约 5–10 分钟。

```bash
cmd.exe /c "cd /d C:\temp_fa_mobile && flutter build apk --release"
```

预期输出：`✓ Built build\app\outputs\flutter-apk\app-release.apk (XX.XMB)`

如果报 `Timeout of 120000 reached waiting for exclusive access to file: ...gradle-8.14-all.zip`：
- 先删掉损坏的缓存：`cmd.exe /c "rmdir /s /q C:\Users\<用户名>\.gradle\wrapper\dists\gradle-8.14-all"`
- 确认 `gradle-wrapper.properties` 用的腾讯镜像，不是 `services.gradle.org`
- 重试第五步

## 第六步：按规范命名并复制产物

```bash
# 从 version 和 build_number 文件读取版本号
VERSION=$(cat zsx-build-deploy/version)
BUILD=$(cat zsx-build-deploy/build_number)
DATE=$(date +%Y%m%d)

# 复制 APK 到 zsx-build-deploy/
cp /mnt/c/temp_fa_mobile/build/app/outputs/flutter-apk/app-release.apk \
   "zsx-build-deploy/frances-allen-v${VERSION}-build${BUILD}-${DATE}.apk"

# 递增 build_number
echo $((BUILD + 1)) > zsx-build-deploy/build_number
```

产物示例：`zsx-build-deploy/frances-allen-v1.2.0-build23-20260531.apk`

## 命名规范

```
<项目>-v<主版本>.<次版本>.<修订>-build<序号>-<日期>.apk
```

| 段 | 示例 | 说明 |
|----|------|------|
| 项目 | `frances-allen` | 固定，小写 kebab-case |
| 版本 | `v1.2.0` | 语义化版本，v 前缀必写 |
| 构建号 | `build23` | 单调递增，每次构建 +1 |
| 日期 | `20260531` | UTC+8，YYYYMMDD 格式 |

## 版本号升级规则

| 改动类型 | 升哪个 | 举例 |
|----------|--------|------|
| 新功能、新页面 | MINOR（中间） | v1.1.0 → v1.2.0 |
| Bug 修复 | PATCH（最后） | v1.1.0 → v1.1.1 |
| 大改架构 | MAJOR（最前） | v1.x.x → v2.0.0 |

升级版本号时编辑 `zsx-build-deploy/version` 文件，build_number 不清零。

## 安装到手机

把 APK 文件传到手机上，直接点击安装即可。安装后桌面显示的应用名为「知识库」，图标为 knowbase.png 的图案。
