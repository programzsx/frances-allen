## 四段表设计理论

一是基础字段。基础字段是说每张表都必备的字段。比如id。比如`create_time`。比如`update_time`。我初步想来。这三个字段是每张表都必备。还有没有其他。我不知道。但是用这个例子可说明基础字段含义是啥。

四是关联字段。这也很常见。比如关联一个标签。标签是独立设计的表。可以有关联的标签表的id。这个也很好理解。

### 问题记录

#### Flutter Web 图片预览 XMLHttpRequest 报错

**问题现象**：

浏览器控制台出现以下错误：
```
Error processing XMLHttpRequest response: InvalidStateError: Failed to read the 'responseText' property from 'XMLHttpRequest': The value is only accessible if the object's 'responseType' is '' or 'text' (was 'arraybuffer').
```

**原因**：

Flutter Web 底层使用 `XMLHttpRequest` 加载图片时，偶尔会出现 `responseType` 类型混淆。框架内部在处理某些请求时将 `responseType` 设置为 `arraybuffer`，但后续读取响应时又尝试以 `text`/`responseText` 方式读取，导致浏览器报错。这是 Flutter Web 框架层面的已知问题，与业务代码无关。

**处理方法**：

只要图片**能正常显示**，此错误可忽略，是调试信息，不影响功能。如果图片加载失败，可尝试：
- 使用 Edge 浏览器替代 Chrome
- 使用 `flutter run -d edge` 启动前端

## 练习模式功能设计

### 三种练习模式

练习页面提供三种模式，通过顶部三栏选择器切换：

#### 1. 随机模式（全局随机）

- 从全部题库（`kb_qa` 全表）中随机抽取 10 道题目
- 无需额外配置，点击"开始练习"即可开始

#### 2. 题库模式（专项练习）

进入题库模式后，提供以下配置项：

- **题库选择器**：DropdownButton，可选择"全部题库"或指定某个题库
- **练习方式**：通过 ChoiceChip 组选择
  - **顺序练习**：按 `random_int` 字段升序取题，保证题目顺序固定
  - **随机练习**：在选定题库中随机抽取（`ORDER BY RAND()`）
  - **错题练习**：在选定题库中筛选错题

#### 3. 错题模式（针对性练习）

- 从全局错题中筛选题目
- **最小错误次数滑块**：Slider 控件，范围 1-10，默认值为 1
  - 默认（最小错误次数=1）：筛选 `wrong >= 1` 的所有错题
  - 设置为 3：筛选 `wrong >= 3` 的题目
  - 可自定义任意值

### 后端API

新增两个题目获取接口：

- `GET /api/qas/sequential/list?limit=10&bank_id=&offset_id=` — 顺序取题，按 `random_int` 升序
- `GET /api/qas/wrong/list?limit=10&bank_id=&min_wrong=1` — 错题筛选，`wrong >= min_wrong`

原有的 `GET /api/qas/random/list` 继续用于随机模式。

### DAO层实现

- `sequential_query()`：`ORDER BY random_int ASC`，支持 `offset_id` 跳过已答题目
- `wrong_query()`：`WHERE wrong >= min_wrong ORDER BY wrong DESC`

### 答题交互（不变）

答题交互逻辑保持不变：填空作答 → 提交 → 精确匹配 → 反馈（答对绿色/答错红色+正确答案） → 上下题导航。

## 富文本标记与渲染

### 标记语法

在题目文本中支持以下标记语法（类 Markdown）：

| 标记 | 语法 | 示例 | 说明 |
|------|------|------|------|
| 加粗 | `**文字**` | `**重点**` | 选中文本点击加粗 → `**XX**`，未选中点击 → `****` 光标在中间 |
| 高亮 | `==文字==` | `==注意==` | 选中文本点击高亮 → `==XX==`，未选中点击 → `====` 光标在中间 |
| 代码 | `` `文字` `` | `` `代码` `` | 选中文本点击代码 → `` `XX` ``，未选中点击 → `` `` `` 光标在中间 |
| 分割线 | `----` | 单独一行 | 点击分割线按钮 → 自动换行插入 `----` 再换行 |
| 填空 | `___` | 保持不变 | 原有功能 |
| 图片 | `![url]` | `![https://...]` | 单独一行，渲染为全宽图片，点击可全屏预览 |

### 渲染样式

题目渲染时（题目列表、练习页面、预览对话框），遇到上述标记按以下样式渲染：

| 标记 | 渲染效果 |
|------|----------|
| `**文字**` | 文字渲染为蓝色（`#0052D9`），加粗 |
| `==文字==` | 添加黄色背景底色（`#FFF9C4`） |
| `` `文字` `` | 添加灰色背景底色（`#F0F0F0`），圆角，文字变红色（`red.shade700`），等宽字体 |
| `----` | 水平分割线（Divider），灰色 |
| `___` | 填空占位符，紫色背景底色（`#E8DEF8`），下划线样式 |
| `![url]` | 全宽图片，圆角，点击全屏预览（InteractiveViewer） |

### 实现方式

创建共享组件 `QuestionRichText`（`lib/pages/question_rich_text.dart`），接收参数：
- `text`: 题目原始文本
- `revealed`: 是否已揭示答案
- `answers`: 答案列表
- `fontSize`: 字号

内部通过两阶段解析：
1. 按换行符分割为块（Block），识别分割线行和图片行
2. 对文本行进行词法分析（Tokenize），识别加粗、高亮、代码、填空标记

使用场景：
- `QaPage`（题目列表卡片）
- `PracticePage`（练习页面题目展示）
- `QaFormPage`（预览对话框）

### 已知问题与修复

#### DropdownButton 断言错误（红色闪屏）

**现象**：点击题目进入编辑页面时，页面一闪而过出现红色错误页面，随后恢复正常。

**原因**：`QaFormPage` 中的 `DropdownButtonFormField`（题库选择器）在 `_banks` 元数据异步加载完成前就被渲染。此时编辑模式下 `_bankId` 已有值（从传入的题目对象中读取），但下拉列表的 `items` 还是空的。Flutter 的 `DropdownButton` 断言要求 `value` 必须是 `items` 中的一个，或者 `value` 为 `null`。两者都不满足时触发红色断言错误。

**修复方案**：在 `_banks` 加载完成前显示 `CircularProgressIndicator` 占位符，不渲染 `DropdownButtonFormField`。标签选择器同样采用此方案，以 `_banks.isEmpty` 作为元数据加载中的判断条件。

```dart
_banks.isEmpty
    ? const CircularProgressIndicator()
    : DropdownButtonFormField<String>(
        value: _bankId,
        items: [/* 题库列表 */],
        // ...
      )
```

#### 题目列表卡片溢出（黄黑条纹）

**现象**：题目列表卡片底部出现黄黑斜纹条纹，控制台报 `RenderFlex overflowed`。

**原因**：原实现使用 `ListTile` + `ConstrainedBox` 嵌套 `QuestionRichText`，`ConstrainedBox` 的固定高度约束与 `QuestionRichText` 内部的多行富文本内容冲突，导致溢出。

**修复方案**：去掉 `ListTile`，改用自定义 `InkWell` + `Padding` + `Row/Column` 布局，让 `QuestionRichText` 自由布局不限制高度。

## 题库层级管理

### 需求背景

`kb_bank` 表存在 `parent_id` 字段，题库支持层级关系（父题库 → 子题库 → 孙子题库）。但前端页面以平铺列表展示，无法体现层级关系；新增/编辑题库对话框中"父题库"字段是文本输入框而非选择器，用户体验不合理。

### 需求描述

1. **层级展示**：题库列表按层级关系缩进展示，根节点靠左，子节点逐级缩进，清晰体现父子关系
2. **父题库选择器**：新增/编辑题库对话框中，"父题库"改为下拉选择器，从已有题库中选择
3. **防止循环引用**：编辑题库时，不能将当前题库自身或其子孙题库设为父题库
4. **根节点标识**：无 `parent_id` 的题库为根节点，显示在列表最左侧
5. **子节点标识**：有 `parent_id` 的题库显示前缀缩进（如 `└─` 或 `  ├─`）

### 实现思路

#### 后端

- `GET /api/banks/tree` 端点已存在（`kb_bank.py:45`），返回嵌套树形结构
- 前端新增/编辑对话框通过 `ApiService.getBankTree()` 获取树数据，填充下拉选项
- 列表页改用 `ApiService.getBanks()` 获取全量数据，前端本地按 `parent_id` 构建树并展平为缩进列表

#### 前端

- **BankPage 列表**：
  1. `_loadBanks()` 改为调用 `ApiService.getBanks()` 获取全量数据
  2. 前端构建 `Map<String, List<KbBank>>`（parentId → children）
  3. 从根节点（`parentId == null`）开始递归遍历，展平为带 `depth` 字段的一维列表
  4. 渲染时每行按 `depth` 缩进（`Padding(left: depth * 24.w)`），前缀显示树形符号（`├─` / `└─` / 空格）

- **父题库选择器**：
  1. 用 `DropdownButton<String?>` 替换 `TextField`
  2. `items` 从 `getBankTree()` 返回的树形结构递归展平生成（带缩进空格）
  3. 编辑模式下排除当前节点及其子孙节点，防止循环引用
  4. 第一项为"无（根题库）"，值为 `null`
