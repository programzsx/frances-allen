# 练习页 · 题库选择区域重构

> 日期：2026-05-26

## 1. 问题

当前题库列表使用 `Wrap` + `_buildBankChip`，题库名称以 chip 形式横向排列再换行，视觉效果杂乱，不够规整。

## 2. 目标

题库名称每行一个，竖排整齐，支持逐层下钻 + 面包屑导航。

## 3. 交互流程

```
根层级（全部题库）
├── 📁 文学          ▶  点击 → 进入文学子层级
├── 📁 历史          ▶
└── ...

面包屑： 全部 > 文学
├── 📁 格言          ▶  点击 → 继续下钻
├── 📁 宋词          ▶
└── ...

面包屑： 全部 > 文学 > 宋词
├── 📄 宋词精选   3题   点击 → 选中题库
├── 📄 宋词名句   5题
└── ...（无更多子层级）
```

## 4. 布局规格

### 4.1 面包屑
- 使用 `Wrap` 组件，自动换行
- 根节点固定显示"全部"
- 每级用 `>` 箭头分隔
- 最后一级高亮（红色实心），显示了选中背景
- 点击任意节点回退到对应层级

### 4.2 题库行
- 每行占满宽度
- 高度 64px
- 左侧：圆形图标区（44×44，浅红背景 + folder/book 图标）
- 中间：题库名称（15sp 粗体）+ 题目数量（12sp，红色如果 >0，灰色如果 0）
- 右侧：
  - 有子题库 → `>` 箭头，表示可下钻
  - 无子题库 → ▶ 播放图标，表示可直接选中练习

### 4.3 空状态
- 当前层级无题库 + 非根层级 → 显示"此层级下无子题库"
- 根层级且总题库为空 → "暂无题库，请先创建"

### 4.4 选中反馈
- 当前层级非根时，底部显示"选择「xxx」"按钮（浅红背景）
- 已选中题库时显示选中的题库名 + 题目数 + 可清除

## 5. 数据加载容错

`_loadBankTree()` 改为独立请求，`getBankTree()` 和 `getDescendantCounts()` 各自 try/catch，一个失败不影响另一个。

## 6. 错题模式 · 错误次数输入

### 6.1 设计初衷

用户输入一个数值 N（例如 3），筛选出"错误次数 ≥ N"的题目进行针对性练习。

### 6.2 交互规格

- 组件：`TextField` 数字输入框，替代原 Slider
- 位置：选择"错题"模式后，在模式选择器下方展示
- 默认值：1
- 校验规则：
  - 不能为空
  - 必须 ≥ 1（不能输入 0 或负数）
  - 输入非法时：输入框变红 + 下方显示提示
  - 输入为空或非法时："开始练习"按钮禁用
- 提示文字："筛选错误次数 ≥ N 的题目"

### 6.3 变量

- `_minWrongCount`：int，默认 1，最小值 1
- 过滤逻辑：`q.wrong >= _minWrongCount`

### 6.4 依赖说明

当前后端 schema 使用 `score`（-1/0/1）而非 `wrong` 计数。需后端支持 `wrong` 字段累加后才能生效。`KbQa` 模型已保留 `wrong` 字段（默认 0）。
