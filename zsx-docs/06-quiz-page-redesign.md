# 答题页面重构

> 日期：2026-05-26

## 1. 问题

当前答题页面布局混乱：
- 题目序号和退出按钮塞在题目卡片内部，不直观
- 没有顶部导航栏，视觉层级不清
- 提交答案后只显示对错，不显示历史练习统计

## 2. 目标

### 2.1 顶部导航栏

```
┌─────────────────────────────────────────┐
│  ✕    第 3 / 20 题           [中国历史]  │  ← AppBar
│  ████████████░░░░░░░░░░░░░░░░░░  15%   │  ← 进度条
└─────────────────────────────────────────┘
```

- 左侧：退出 ✕ 按钮
- 中间：`第 N / M 题`
- 右侧：题库名称 chip（如有选中题库）
- 下方：进度条 + 百分比

### 2.2 答题区域（不变）

- 题目卡片（序号 + 题目类型 + 题目内容）
- 答题区（输入框 + 提交按钮）
- 结果区（对错标记 + 正确答案）

### 2.3 历史统计卡片（新增，提交答案后显示）

```
┌─────────────────────────────────────────┐
│  练习统计                                │
│  ┌──────┐  ┌──────┐  ┌──────┐          │
│  │  12  │  │   8  │  │   4  │          │
│  │总次数│  │ 答对 │  │ 答错 │          │
│  └──────┘  └──────┘  └──────┘          │
│            正确率 67%                    │
└─────────────────────────────────────────┘
```

- 三个统计数字：total / right / wrong（含本次回答）
- 正确率百分比
- 统计数字从 `_questions[_currentIndex]` 读取（_submitAnswer 已本地更新）

### 2.4 底部导航栏（不变）

- 上一题 / 题目序号 / 下一题（或返回）
- 退出确认栏

## 4. 滑动切换题目

### 4.1 交互

- 使用 `PageView` 替代按钮导航
- 左滑 → 下一题，右滑 → 上一题
- 切换流畅（300ms 动画）

### 4.2 底部页面指示器

- 圆点样式
- 当前页：红色宽点（20×8）
- 已答页：浅红圆点（8×8）
- 未答页：灰色圆点（8×8）

### 4.3 最后一题提示

- 答完最后一题后，底部显示"已完成全部题目，左滑回顾"

## 3. 统计更新逻辑

`_submitAnswer()` 修改：
- API 调用静默失败（不弹 SnackBar）
- 本地数据始终更新（total/right/wrong 本地 +1）
- KbQa 模型已有 total/right/wrong 字段（默认 0）
